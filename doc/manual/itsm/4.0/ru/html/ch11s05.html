Матрица "Критичность - Влияние - Приоритет"

OTRS::ITSM предполагает пять уровней приоритета для заявок:

  • [ Criticality ] - Критичность

    Значимость ("критичность") сервиса для ИТ пользователя(ей)/клиента(ов)

  • [ Impact ] - Влияние

    Влияние сбоя соответствующего сервиса на работу пользователя(ей)/клиента(ов)

  • [ Priority ] - Приоритеты

    Приоритет в OTRS::ITSM есть результат, зависящий от пары критичность - влияние

Приоритет заявки в OTRS::ITSM определяется по матрице показанной ниже и результат приоритезации используется в обзорах очередей.
