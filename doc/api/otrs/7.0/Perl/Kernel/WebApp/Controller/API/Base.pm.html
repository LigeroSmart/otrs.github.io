NAME

Kernel::WebApp::Controller::API::Base - Base class for all API endpoints.

PUBLIC INTERFACE

Provides endpoint-level version information based on Kernel::WebApp::Controller::API::Role::HasEndpointVersion.

Handler()

implements a 4-phase request handling. Endpoints can provide their own implementation of these phase methods:

"Authenticate()"
"DataValidate()"
"Authorize()"
"Action()"

The "Handler()" method will execute the 4 phases in this order. If any of these create a response (e. g. 401 Unauthenticated) or return a false value, the processing is stopped.

The HTTP PATCH method has a special meaning here. It serves as a "pre-flight check" that only executes the first 3 verification phases, but not the final "Action()" phase. If the verification was ok, a 200 OK response is sent. This can be used for validation purposes on any API endpoint.

Authenticate()

checks if the user has the needed credentials (e. g., a customer access token).

Default implementation is a no-op, can be overridden by endpoints or roles.

DataValidate()

checks if the data payload is valid (e. g. a form was correctly filled).

Default implementation is a no-op, can be overridden by endpoints or roles.

Authorize()

checks if the authenticated user has the necessary permission for the requested resource(s).

Default implementation is a no-op, can be overridden by endpoints or roles.

Action()

after all previous checks succeeded, this method will perform the requested action (e. g. create or fetch a ticket).

Default implementation is a no-op, can be overridden by endpoints or roles.

RequestPath()

this method will just set the url path for this controller.
